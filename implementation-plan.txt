📋 分阶段实施计划
🟢 阶段 1 — 核心功能（纯前端，GitHub Pages）
目标： 用户填写表单 → 生成并下载 .fit 文件
技术细节：

React + Vite 项目
使用官方 @garmin/fitsdk 的 Encoder 类
写入 weight_scale 消息类型，包含所有字段（体重、体脂、肌肉量、骨量、体水分、内脏脂肪、BMI）
触发浏览器下载 .fit 文件
部署：gh-pages 分支，GitHub Actions 自动构建

预计时间： 1-2小时

🟡 阶段 2 — 直接上传（Python 后端）
目标： 用户输入 Garmin 账号 → 自动上传，无需手动导入
技术细节：

FastAPI 接收：{ fit_bytes, email, password }
使用 python-garminconnect 进行 OAuth 认证
调用 upload_fit_file() 上传
凭据绝不存储，用完即丢
HTTPS 加密传输
部署到 Railway 免费套餐

预计时间： 2-3小时

🛠️ 关键技术片段预览
前端：生成 .fit 文件
jsimport { Encoder, Profile } from "@garmin/fitsdk";

const encoder = new Encoder();

encoder.onMesg(Profile.MesgNum.FILE_ID, {
  type: "weight",
  manufacturer: "development",
  timeCreated: new Date(),
});

encoder.onMesg(Profile.MesgNum.WEIGHT_SCALE, {
  timestamp: new Date(),
  weight: 78.5,          // kg
  percentFat: 18.5,      // %
  percentHydration: 55.0,
  muscleMass: 35.0,      // kg
  visceralFatRating: 10,
  bmi: 24.2,
});

const fitBytes = encoder.close();
// → Blob → descargar como .fit
后端：subir a Garmin
pythonfrom garminconnect import Garmin

async def upload(email, password, fit_bytes):
    client = Garmin(email, password)
    client.login()
    client.upload_fit_file_from_bytes(fit_bytes)
    # token se descarta inmediatamente
